üöÄ Using This in Production

Two Deployment Options

This demo shows both the REST API and optional web interface. Choose what fits your needs:

Option A: API Only (Recommended for most users)

Deploy just the FeatureServer REST API. Your users consume it via their own ArcGIS tools.

# Deploy without demo pages
export SERVE_DEMO_PAGES=false
npm start

Option B: Full Solution (What you're seeing now)

Deploy the API plus these demo web pages. Great for showcasing capabilities.

# Deploy with demo pages
export SERVE_DEMO_PAGES=true
npm start

üí° Note: Most production deployments use Option A. The demo pages are just examples showing how clients would consume your API.
